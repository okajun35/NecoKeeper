# ハッカソン向け計画サマリ

## デフォルトロケール環境変数化プラン
- デプロイごとに初期言語を制御する仕組みの検討メモ。
- `docs/default-locale-plan.md` に詳細を保管し、現在は実施保留。

## ハッカソン優先実装プラン
- **デプロイ整備**: Render/Fly/Railway想定の設定ファイルとREADME、サンプル`.env`を整備し、再現性の高い導入手順を用意する。
- **PWAオフライン対応**: Publicフォームでオフライン入力→再同期が成立するよう、Service Workerと同期処理を仕上げる。
- **QR→Public体験磨き**: QRカード出力テンプレや履歴表示を整え、QR読み取りから記録保存までのデモ導線を滑らかにする。
- **ダッシュボード可視化**: 猫ステータスや記録推移を即座に把握できる簡易チャートを追加し、価値訴求を強化する。
- **AI/OCRスパイス**: 軽量なAI活用（自動タグ付けやOCR補助など）を組み込み、ハッカソン向けのインパクトを演出する。

## AIデモスパイスプラン
- **Kiro Hook / CLI**: `media/inbox/`を監視してOCR処理を呼び出す仕組みを用意し、ローカルでもデモでも使える導線にする。
- **スクリプト層**: `scripts/ocr_hook.py` から `ai_utils/ocr_client.py` を呼び分け、TesseractやクラウドOCRを環境変数で切り替える。
- **サービス層**: OCR結果や自動タグを `app/services/ai_ingest.py` で下書きデータとして保存し、`paper_transfer_flag` 付きで扱えるようにする。
- **UI連携**: 管理画面ダッシュボードに最新取り込みカードを追加し、OCRで補助された記録が確認できる体験を提供する。

## MCPベース自動登録プラン
- **サーバ素体**: `modelcontextprotocol` のFastMCPで `app/mcp/server.py` を作成し、ローカルから起動できるMCPサーバを用意する。
- **ツール定義**: `register_cat`（猫プロフィール登録）、`generate_qr`（QR画像生成）などのツールを作り、既存サービス/APIを呼び出す。
- **認証設計**: 内部サービス呼び出しとREST API経由の両案を検討し、サービスアカウントやAPIキーで安全に自動化できるようにする。
- **運用手順**: `.kiro/settings` にサーバコマンドを登録し、READMEにMCP利用手順とデモシナリオ（自動登録→QR取得）を追記する。
